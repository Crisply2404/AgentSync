# UI

## 目的
让你用“点点点”的方式完成配置与同步，不需要你手写路径和命令。

## 模块概述
- **职责:** 配置界面、选择文件夹、展示同步进度与日志
- **状态:** 🚧开发中
- **最后更新:** 2026-02-14

补充说明（更贴近你看到的界面）：
- 顶部是 Tab 导航（传送/向导/内容/目标/历史）
- 中间是单列卡片内容区（更像原型那种“干净舒服”的布局）
- 同步开始后会在顶部显示“同步中”提示，你切到其它 Tab 也不会中断（同步在后台跑）

## 规范

### 需求: 一键同步（单向推送）
**模块:** UI  
用户从源电脑点击“一键同步”，完成后能看到结果摘要和日志入口。

#### 场景: 连接未配置
前置条件：Host/User/Key 未填完整  
- 预期结果：按钮不可用或点击后提示“先完成连接设置”

#### 场景: 同步进行中
前置条件：同步任务已启动  
- 预期结果：展示进度（当前同步项 + 传输状态），切换页面不影响同步，结束后给出摘要

### 需求: 设备向导（SSH 模式，目标端免安装）
**模块:** UI  
第一次使用时，用户不想研究 SSH：希望按向导做完一次性配置后就能测试连接并开始同步。

#### 场景: 没有 SSH 密钥
前置条件：未选择私钥路径  
- 预期结果：页面提供“一键生成/复用专用密钥”，并能显示公钥

#### 场景: 需要在目标端授权
前置条件：已拿到公钥、已填目标端目录  
- 预期结果：页面生成“一次性初始化命令”，用户可复制到目标端执行
 - 预期结果：如果不方便“从本机复制发过去”，可生成局域网分享链接，在目标端浏览器打开后复制

#### 场景: 目标端执行完成后回到本机
前置条件：目标端已执行初始化命令  
- 预期结果：用户可以在向导页直接点“测试连接”验证是否可用
